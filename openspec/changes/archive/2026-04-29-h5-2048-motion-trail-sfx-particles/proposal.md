## Why

当前方块位移主要靠 CSS Grid 过渡配合固定时长与近似匀速曲线，运动感偏「平移到位」而非「加速滑行」；规格要求拖尾与闪耀粒子，但棋盘层仍以通用 shimmer + 合并 burst 为主，缺少沿滑动方向、贴近格面摩擦的碎屑感与可辨拖尾；全无音效，合并与结局缺乏反馈。

## What Changes

- **位移动画**：为格子过渡引入**非线性缓动**（可见的起步加速 / 停靠减速，或等效曲线），避免观感上「匀速换格」或「瞬切」。
- **拖尾**：在位移阶段渲染**沿滑动方向的轨迹类拖尾**（条带或淡出尾迹），随该轮动画结束清除。
- **摩擦碎屑**：在方块移动阶段于棋盘区域内生成**与地面摩擦语义一致的细小粒子**（颜色与密度可控），与合并闪耀区分。
- **音效**：增加 **移动 / 合并** 短期音效；**游戏结束**时区分 **失败** 与 **打破最佳分（新纪录）**（若当局刷新最佳）；音频文件随静态构建打包，路径兼容 itch.io `base`。

### Modified Capabilities

- **`h5-2048-game`**：在位移动画、拖尾与粒子语义上收紧或增补需求；新增音效与静态资源约束相关条目。

## Impact

- **前端**：`Tile.vue` / `Board.vue`（过渡曲线、传入滑动方向或向量供 FX）；`FxCanvas.vue` 或拆分小型绘制模块（拖尾 + 摩擦粒子）；`useGame2048.ts` / `App.vue`（音效触发时机、`lost` / `best` 与合并事件）。
- **资源**：新增 `public/` 或 `src/assets/` 下短音频（格式以实现为准，如 `*.mp3` / `*.ogg`），`vite` 构建与 `base` 相对路径自检。
- **性能**：粒子与拖尾 SHALL 沿用池化或数量上限；音频预加载与移动端自动播放策略在设计中说明。
