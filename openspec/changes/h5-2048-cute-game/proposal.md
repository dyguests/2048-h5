## Why

移动端 H5 上缺少一款视觉与反馈都足够「可爱」的 2048：玩家需要清晰的滑动反馈、合并时的愉悦感，以及移动过程中丰富的动效（粒子与拖尾）来提升可玩性与品质感。本变更在仓库内新增独立 H5 实现，作为可演示、可扩展的前端小游戏。

## What Changes

- 新增基于 Web 的 2048 核心玩法：4×4 网格、方向滑动合并、生成新方块、胜负判定与重开。
- 采用**可爱风** UI（圆角、柔和配色、可选轻量角色/表情或数字样式），适配常见移动浏览器视口与触摸。
- **滑动/移动过程**使用显式动画（非瞬时跳格），让方块从旧格平滑过渡到新格。
- 动画期间叠加**闪耀粒子**效果（合并或移动高峰时增强反馈）。
- 移动的 tile 带**运动拖尾**（轨迹或渐变尾迹），强化方向感与速度感。
- 提供基础工程结构（构建/入口、静态资源位），便于后续接入 CI 或部署。

## Capabilities

### New Capabilities

- `h5-2048-game`: 2048 H5 游戏的行为与体验要求：核心规则、可爱风视觉规范、滑动与合并动画、粒子与拖尾效果、输入与可访问性基线。

### Modified Capabilities

- （无：当前 `openspec/specs/` 下无既有能力需变更。）

## Impact

- 新增前端页面、样式与动画相关资源；可能引入轻量动画/粒子实现（纯 CSS/Canvas 2D/SVG 或现有构建栈内库，由设计阶段定稿）。
- 无既有 API 契约变更；若未来单独部署，仅需静态托管或现有站点的子路径。
